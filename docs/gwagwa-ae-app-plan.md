# 과학과애(愛) 프로그램 앱 기획안

> 2026 국립과천과학관 성인과학학습 프로그램 \<과학과애\> 운영을 위한 웹앱 계획.
> 기존 갈다 생태도감 웹앱(eco-explorer, https://galdar-eco-explorer.vercel.app)을 확장하는 방향으로 설계.

---

## 1. 배경과 목표

제안서 기준 과학과애는 4개 분야로 운영된다.

| 분야 | 차시 | 장소 | 앱에서의 역할 |
|---|---|---|---|
| 시민과학 | 16차시+ (가족동반) | 과학관 주변/우리 동네 | **생태도감 웹앱으로 탐사 기록 축적** (제안서에 명시) |
| 실험과학 | 30차시+ | 과학관 내 (패밀리창작놀이터) | 스탭 현장기록 → 블로그/인스타 자동 포스팅 |
| 예술과학 | 18차시+ | 과학관 내 | 스탭 현장기록 + 참가자 결과물 갤러리 |
| 현장과학 | 10차시+ | 전국 탐방 (나로우주센터, 예미랩 등) | 기존 eco-explorer의 이벤트 모델 활용한 탐방 기록 |

앱이 풀어야 할 문제 (제안서의 2025 개선과제와 과업 연결):

1. **현장과학의 일회성 이벤트화** → 탐방 기록이 앱에 축적되어 과학관 본관 경험(과학과애 라운지/갤러리)과 연결되도록.
2. **블로그 후기 월 2건+, 후기 영상 월 1건, 과천과학관·갈다 채널 동시 업로드** 의무 → 스탭이 사진+메모만 올리면 포스팅 초안이 자동 생성되는 파이프라인으로 제작 부담 최소화.
3. **참여자의 생생한 후기 노출을 통한 홍보** → 참가자 기록 갤러리가 곧 홍보 자산이 되도록.

접수/결제/수료증은 과학관 LMS가 담당하므로 앱 범위에서 제외한다. 앱은 "기록·축적·발행" 도구다.

## 2. 참가자용 (로그인 없음 — 현행 editToken 방식 유지)

### 2.1 공통

- **프로그램 안내 홈**: 월별 일정 캘린더, 분야별(시민/실험/예술/현장) 필터, 각 프로그램 상세 페이지(기존 Event guidePages 3페이지 구조 재활용). LMS 신청 링크로 연결.
- **QR 진입**: 각 회차 현장에 QR 부착 → 해당 회차 페이지로 바로 진입(기록 업로드, 자료, 설문).
- **기록 갤러리**: 분야별/회차별 모아보기. 좋아요(기존 Like 모델), 내 기록 수정·삭제(기존 OwnerControls).
- **설문**: 회차 종료 화면에서 설문 링크 노출(문자 발송 보완 채널).

### 2.2 시민과학 — 생태도감 (기존 기능 거의 그대로)

제안서에 이미 명시된 활용 방식: "사진 업로드 → AI가 생물 종류와 장소 확인 → 다양한 도감 뷰(모아보기/선택보기/지도보기)".

- 현행 submit 플로우(사진 → EXIF GPS → Gemini 분석 → 도감 등록) 유지.
- **과천 지역 생태도감**으로 기본 이벤트 설정. 세부 프로젝트(매미 지도, 우리동네 새 지도, 외래생물 지도, 자원발굴단, 오염지킴이)를 각각 이벤트/phase로 등록해 프로젝트별 지도·랭킹 제공.
- 위치 카테고리를 과천 환경에 맞게 조정(갯벌/바다 → 하천/공원/산림/주택가 등).
- 가족동반 프로그램이므로 팀 이름 = 가족 이름 그대로 활용 가능.

### 2.3 현장과학 — 탐방 기록 (기존 이벤트 모델 확장)

탐방 1회 = 이벤트 1개. 회차당 20명 내외 단체버스 이동이라는 운영 형태에 맞춤.

- **사전(사이언스 프리퀄)**: 방문지 소개 자료, 사전 질문 제출 폼(버스에서 과학커뮤니케이터가 답하는 소재로 활용).
- **현장**: 사진+메모 업로드(오프라인 대비: 통신 불량 지역이 많으므로 업로드 실패 시 재시도 큐 필요), 간단한 현장 미션(예: "오늘의 발견 3가지").
- **사후**: 탐방별 공유 갤러리 → 과학과애 라운지 디스플레이/알림판 콘텐츠로 재사용. 다음 연계 프로그램 추천 노출(입문→심화 연결 전략 지원).

### 2.4 실험·예술과학 — 결과물 갤러리 (선택 참여)

- 참가자가 자기 결과물(바이오아트, 드로잉, DNA 키링 등) 사진을 회차 갤러리에 업로드. AI 분석은 생물 인식 대신 "작품 설명 보조" 정도로 단순화하거나 생략.
- 과학관 드로잉(매월 개설)처럼 연속 프로그램은 회차별 phase로 누적 기록 → 참여자의 "작은 성장" 가시화.

## 3. 스탭용

### 3.1 핵심: 현장기록 → 블로그/인스타 자동 포스팅 파이프라인

스탭의 입력은 딱 두 가지: **사진 여러 장 + 짧은 메모**.

```
[기록]  회차 선택 → 사진 n장 업로드 → 메모 입력(하이라이트, 참여자 반응, 강사 멘트 등)
        → 초상권 동의 확인 체크(미체크 시 발행 단계 진입 불가)
[생성]  AI(Claude API)가 프로그램 메타데이터(프로그램명·강사·일시·내용 소개) + 메모 + 사진을 받아
        ① 네이버 블로그용 후기 초안: 제목, 본문(사진 배치 위치 표시 포함), 800~1200자
        ② 인스타그램 캡션: 후킹 문장 + 본문 + 해시태그(#과학과애 #국립과천과학관 #갈다 + 프로그램별 태그)
        를 각각 과천과학관 채널 톤/갈다 채널 톤으로 생성
[검토]  스탭이 초안 검토·수정 → 승인 (AI 생성물은 반드시 사람 검토 후 발행)
[발행]  채널별 발행 — 아래 단계적 적용
```

**발행 자동화의 현실적 단계** (외부 API 제약 때문에 3단계로 나눔):

1. **1단계 — 반자동 (확실, 즉시 가능)**: 승인된 초안을 원클릭 복사(블로그 본문/인스타 캡션) + 리사이즈된 사진 묶음 다운로드. 스탭은 붙여넣기만 하면 됨. 포스팅 1건 작업이 30분+ → 5분 이내로 단축.
2. **2단계 — 인스타그램 자동 발행**: Meta Graph API의 Content Publishing 사용. 갈다 인스타 계정을 비즈니스/크리에이터 계정으로 전환하고 Facebook 페이지에 연동해야 함. 과천과학관 공식 계정은 권한 문제로 1단계 방식 유지 가능성 높음.
3. **3단계 — 네이버 블로그**: 네이버 공식 블로그 글쓰기 API는 신규 지원이 사실상 중단된 상태라 자동 발행은 보장 불가. 반자동(검토 후 붙여넣기)을 기본으로 하되, API 가용 여부를 착수 시점에 재확인.

발행 상태(초안/검토중/발행완료)와 발행 채널·URL을 기록해 **월별 의무 건수(블로그 2건+, 영상 1건) 달성 현황**을 대시보드에서 추적.

### 3.2 회차·프로그램 관리 (기존 admin 확장)

- 프로그램(분야·강사·소개) / 회차(일시·장소·정원) CRUD. 회차별 QR 코드 생성·인쇄.
- 기존 기능 유지: 제출물 숨김/일괄 태깅, 가이드 페이지 편집, 업로드 ON/OFF.
- **권한 개선**: 현행 하드코딩 단일 비밀번호(`galdar123`)를 스탭 개별 계정(이메일 매직링크 수준의 가벼운 인증)으로 교체. 발행 권한은 별도 롤로 분리(기록은 누구나, 발행 승인은 담당자만).

### 3.3 운영 보조

- **초상권 동의 관리**: 회차별 전자 동의 폼(QR로 참가자가 직접 체크) 또는 스탭의 서면 동의 수합 체크. 동의 데이터는 회차에 묶어 보관, 사업 종료 후 파기 절차 지원(제안서 개인정보 조항 대응).
- **통계 대시보드**: 회차별 참여 기록 수, 갤러리 좋아요, 도감 누적 종 수, 포스팅 발행 현황 → 월 2회 과학관 보고와 결과보고서 작성에 바로 활용.
- **현장 체크리스트(선택)**: 안전점검·물품 준비 체크리스트를 회차에 첨부해 모바일에서 확인.

## 4. 데이터 모델 확장 (Prisma)

기존 모델 유지 + 추가:

```
Program        id, category(시민/실험/예술/현장), title, description, instructor, isActive
Session        id, programId, round, dateTime, place, capacity, qrSlug
               // 기존 Event는 시민과학 도감 프로젝트·현장과학 탐방용으로 계속 사용, Session과 1:1 연결 가능
StaffLog       id, sessionId, authorId, memo, mediaUrls(JSON), consentConfirmed(boolean), createdAt
GeneratedPost  id, staffLogId, channel(naver_blog|instagram), tone(museum|galdar),
               title, body, hashtags, status(draft|approved|published), publishedUrl, publishedAt
StaffUser      id, email, name, role(recorder|publisher|admin)
SurveyLink     sessionId, url   // 또는 Session에 필드로
```

`Submission`(참가자 기록)은 그대로 두고 `sessionId` FK만 추가하면 회차별 갤러리가 된다.

## 5. 단계별 로드맵

사업 일정(공모 6월, 정규 프로그램 6~11월) 기준.

| 단계 | 시점 | 내용 |
|---|---|---|
| **Phase 1** | 6월 시민과학 특강 전 | 프로그램 안내 홈 + 과천 생태도감 이벤트 세팅 + 스탭 기록 화면 + AI 초안 생성(1단계 반자동 발행) |
| **Phase 2** | 7~8월 | 현장과학 탐방 이벤트(사전 질문·오프라인 재시도 큐), 인스타 자동 발행(Graph API), 스탭 계정 인증 |
| **Phase 3** | 9월~ | 통계 대시보드(보고 자동화), 실험·예술 결과물 갤러리 고도화, 초상권 전자 동의 폼 |

기술 스택은 현행 유지: Next.js 14 + Prisma/Postgres + Vercel Blob + Gemini(이미지 분석) + **Claude API(포스팅 초안 생성)**.

## 6. 리스크 및 유의사항

- **초상권**: 동의 확인 없이 사진이 발행 단계로 넘어가지 않도록 시스템으로 강제. 얼굴이 식별되는 사진은 AI 초안 단계에서 경고 표시(선택 구현).
- **AI 생성 글**: 항상 사람 검토 후 발행. 강사 이름·기관명 등 고유명사는 프로그램 메타데이터에서 주입해 오기(誤記) 방지.
- **네이버 블로그 자동화 의존 금지**: 반자동을 기본 설계로 두고 자동화는 보너스로 취급.
- **현장과학 통신 환경**: 예미랩(지하), 백두대간 수목원 등은 통신이 불안정 → 업로드 실패 허용 + 로컬 보관 후 재전송 설계 필수.
- **참여자 진입 장벽**: 성인·시니어(꽃중년) 대상이므로 로그인 없는 현행 모델 유지가 강점. QR 한 번으로 끝나는 동선을 모든 화면의 원칙으로.
